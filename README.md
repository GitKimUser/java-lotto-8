📝 기능 목록
입력 기능
[x] 로또 구입 금액을 입력받는다.

[x] 입력값이 숫자가 아닐 경우 예외를 발생시킨다.

[x] 입력값이 1,000원 단위가 아닐 경우 예외를 발생시킨다.

[ ] 당첨 번호를 입력받는다.

[ ] 번호는 쉼표(,)를 기준으로 분리한다.

[ ] 당첨 번호가 6개가 아닐 경우 예외를 발생시킨다.

[ ] 각 번호가 1부터 45 사이의 숫자가 아닐 경우 예외를 발생시킨다.

[ ] 중복된 숫자가 있을 경우 예외를 발생시킨다.

[ ] 보너스 번호를 입력받는다.

[ ] 보너스 번호가 1부터 45 사이의 숫자가 아닐 경우 예외를 발생시킨다.

[ ] 보너스 번호가 당첨 번호와 중복될 경우 예외를 발생시킨다.

핵심 로직 및 도메인 기능
[x] 구입 금액에 해당하는 만큼 로또를 발행한다.

[x] camp.nextstep.edu.missionutils.Randoms.pickUniqueNumbersInRange(1, 45, 6) API를 사용하여 무작위 번호를 생성한다.

[x] 발행된 각 로또 번호는 오름차순으로 정렬한다.

[x] Lotto 클래스를 요구사항에 맞게 구현한다.

[x] 생성자에서 로또 번호 개수가 6개가 아닐 경우 예외를 발생시킨다.

[x] (추가) 로또 번호에 중복된 숫자가 있는지 검증하는 로직을 추가한다.

[ ] (추가) 로또 번호가 1~45 범위를 벗어나는지 검증하는 로직을 추가한다.

[ ] 발행된 로또와 당첨 번호, 보너스 번호를 비교하여 당첨 등수를 결정한다.

[ ] 각 등수별 당첨 개수를 계산한다.

[ ] 5등: 3개 번호 일치

[ ] 4등: 4개 번호 일치

[ ] 3등: 5개 번호 일치

[ ] 2등: 5개 번호 + 보너스 번호 일치

[ ] 1등: 6개 번호 일치

[ ] 총 당첨금을 계산한다.

[ ] 총 수익률을 계산한다.

[ ] 수익률은 소수점 둘째 자리에서 반올림하여 형식에 맞게 문자열로 만든다. (예: "62.5%")

출력 기능
[x] 구매한 로또의 개수를 출력한다.

[x] 발행된 모든 로또의 번호를 형식에 맞게 출력한다.

[ ] 당첨 통계를 형식에 맞게 출력한다.

[ ] 최종 수익률을 형식에 맞게 출력한다.

예외 처리
[x] 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시킨다.

[x] 발생한 예외에 대해 [ERROR]로 시작하는 에러 메시지를 출력한다.

[x] 예외 발생 시, 해당 입력부터 다시 받아서 프로그램을 정상적으로 이어나간다.

클래스 분리 및 객체 설계
[ ] Application: 프로그램의 시작과 전체적인 흐름을 관리한다.

[ ] Lotto: 로또 1장을 의미하는 도메인 객체 (제공된 클래스 활용 및 기능 추가).

[ ] InputView: 사용자의 입력을 받고, 기본적인 문자열 처리를 담당한다.

[ ] OutputView: 게임 결과 및 안내 문구 출력을 담당한다.

[ ] LottoGameController: 입력, 로또 발행, 당첨 결과 계산 등 로또 게임의 핵심 비즈니스 로직을 총괄한다.

[ ] WinningLotto: 당첨 번호와 보너스 번호를 관리하는 객체.

[ ] Rank (Enum): 1등부터 5등까지의 당첨 기준(일치 개수), 당첨금을 관리하는 열거형 타입.

[ ] Validator: 입력값의 유효성 검증 로직을 모아둔 클래스.
